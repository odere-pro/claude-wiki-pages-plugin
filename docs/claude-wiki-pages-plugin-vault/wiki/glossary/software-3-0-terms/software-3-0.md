# Software 3.0

## Definition

The development posture where the repo's docs, tools, design, system design, context, and memory are equally accessible and usable by humans and by agents — enforced by gates, not just by convention. Named after the Software 3.0 paradigm; applied here to make every surface dual-entry by construction.

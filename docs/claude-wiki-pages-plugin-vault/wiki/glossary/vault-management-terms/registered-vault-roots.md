# registered vault roots

## Definition

The set of vault root paths currently enrolled in the vault registry. A vault root becomes registered via the `add` lifecycle command and is eligible to be made the active vault. Alias for the membership dimension of `vault registry`.

## Key Principles

- The set of vault root paths currently enrolled in the vault registry.
- Canonical term in the claude-wiki-pages **Vault management terms** vocabulary; conforms to the project glossary and is enforced by `validate-docs.sh`.

## Examples

- `add`
- `vault registry`

# LLM Wiki — Obsidian Experience

The Obsidian-specific features of the wiki: the graph view with topic islands and color groups, wikilink navigation, the Dataview dashboard, and how the plugin generates `.obsidian/` configuration deterministically.

## Definition

The Obsidian experience is the visual and navigational layer on top of the wiki's structured pages. The plugin generates `.obsidian/app.json` and `.obsidian/graph.json` deterministically so every vault opens with the same graph view configuration.

## Key Principles

**Graph view: topic islands.** The graph view renders each top-level topic folder as a colored cluster (one color per topic). Sources, synthesis, and the root MOC are filtered out of the drawn graph — they stay searchable but do not appear as nodes. The result is a clean island-shaped topology (ADR-0023, ADR-0033, ADR-0036).

**Deterministic config.** `apply-obsidian-config.sh` generates `.obsidian/app.json` and `.obsidian/graph.json`. If these files are lost or mangled, re-running the script rebuilds them. Color groups are regenerated by the polish agent or the graph-colors skill (headless fallback: write `graph.json` directly — a running Obsidian may overwrite it; restart Obsidian after a headless write).

**Link resolution.** Obsidian resolves `[[target]]` by exact vault path > file basename > alias — never by alias alone or by title. The piped-basename convention (`[[file-basename|Title Case]]`) is required for correct navigation. A bare `[[Title Case]]` link is a ghost node (ADR-0032).

**Recommended plugins.** Dataview (for `dashboard.md` queries — required for the dashboard to render). obsidian-wikilink-types (for typed link display — recommended at 50+ pages).

**Path-scoped rules.** The plugin's `rules/` directory contains path-scoped rules that constrain what agents and skills may do. These are not Obsidian features; they are Claude Code rules that operate on the vault.

## Examples

When a new top-level topic folder is created during ingest, the polish agent automatically adds a color group for it in `.obsidian/graph.json`. Without the polish agent, the new topic folder would appear as an uncolored island.

## Related Concepts

The two-layer exclusion model (index exclusion + graph-view exclusion) is specified in ADR-0023. The strict-tree topology (ADR-0036) determines which edges appear within each island. The deterministic Obsidian config guarantee is specified in ADR-0035.

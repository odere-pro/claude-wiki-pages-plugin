## Summary

The `voice` skill defines the house writing voice for the plugin — how to write prose a newcomer wants to read and an engineer can trust. Two registers: explanatory (for learning) and engineer (for building/operating). Defines the LLM-artifact blocklist, vocabulary rules, and how to write a wiki page.

## Key Claims

Covers: Voice Skill, Explanatory Register, Engineer Register, LLM-Artifact Blocklist, Vocabulary Gate, Wiki Page Voice.

LLM-artifact blocklist (cut in both registers): em-dash drama, filler openers ("It's worth noting"), hype adjectives (powerful, seamless, robust, effortless), "not just X it's Y" frame, hedge stacks, over-bolding, echo summaries, robotic triads, provenance as adjective, inline re-definition. Wiki page rules: open with the class definition in plain language; the opening sentence stands alone; stay in the engineer register for the body; never assert an uncited fact; end synthesis/query answers with a `## Sources` heading. The aloud test: if it sounds like a person who understands the system explaining it on purpose, ship it.

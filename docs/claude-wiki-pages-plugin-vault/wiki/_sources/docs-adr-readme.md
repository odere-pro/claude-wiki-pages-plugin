## Summary

The ADR README documents the conventions for the `docs/adr/` directory: what an ADR is, what it contains, how status values work (Proposed/Accepted/Superseded/Deprecated), and the append-only amendment rule (a new ADR supersedes rather than edits the old one). It serves as the entry point for the ADR log.

## Key Claims

Status: Standing doc (not an ADR itself). ADRs record one decision each: status, date, SPEC anchor, context, decision, consequences, and test/verification plan. Status lifecycle: Proposed → Accepted → (optionally) Superseded or Deprecated. Amendment rule: a new ADR supersedes an old one; old ADRs are never edited. ADRs are immutable once Accepted. The directory currently holds ADR-0001 through ADR-0036.

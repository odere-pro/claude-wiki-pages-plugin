# Documentation Rules

## Metadata

- **File**: `raw/repo/rules/docs.md`
- **Scope**: `docs/**`
- **Type**: Path-scoped rule file

## Summary

Defines plain Markdown conventions for project documentation files under `docs/`. Forbids Obsidian-flavored syntax (wikilinks, YAML frontmatter) in project docs; mandates standard Markdown links and GitHub-flavored callouts.

## Key Claims

Files in `docs/` are plain Markdown, not Obsidian-flavored. Use standard Markdown links (not wikilinks). No YAML frontmatter required. GitHub-flavored callouts are permitted. Documents must be self-contained and cross-reference other docs by filename.
Covers: Documentation Rules, Path-Scoped Rules, Wiki Rules

# Folder Note

The per-folder index file named exactly after its folder (`wiki/<topic>/<topic>.md`, `type: index`) that serves as the MOC for that branch of the topic tree.

## Definition

A folder note is a markdown file whose filename stem matches its parent directory name, placed inside that directory. In this plugin's vault layout, every topic folder `wiki/<topic>/` contains a file `wiki/<topic>/<topic>.md` with `type: index`. This file is the folder's Map of Content: it lists and links all pages in that folder and its children, grouped by theme, under `## Section` headings.

The folder note is an Obsidian community convention with existing tooling support (notably the "Folder Notes" community plugin). The plugin adopts the name and convention, replacing the legacy `_index.md` form in schema version 3. A legacy `_index.md` is still accepted by the engine but flagged as `legacy-index-filename` at verify time; the engine `migrate --write` command renames it to the folder-note form.

The root index is always `wiki/index.md` — the vault MOC, which does not live inside a topic folder. Every other index page is a folder note.

## Key Principles

**Filename stem equals folder name.** The file `wiki/agents/agents.md` is the folder note for `wiki/agents/`. The `wiki/docs/docs.md` file is the folder note for `wiki/docs/`. This naming rule is enforced by lint and by the `verify` command; a mismatch is a structural error.

**`type: index` always.** Folder notes use `type: index` in frontmatter regardless of the topic they cover. No other type is valid for a folder note.

**`aliases:` for discovery.** Every folder note must carry `aliases:` with slug, title-case, and abbreviation variants of the topic name. These aid search and autocomplete but do not affect link resolution; links still target the basename.

**`children:` and `child_indexes:` in piped wikilink form.** Every direct page child appears in `children:` and every direct subfolder's folder note appears in `child_indexes:`, both as quoted piped wikilinks (`"[[page-basename|Page Title]]"`). A plain title string produces no graph edge and is a lint error.

**Body groups children by theme.** The folder note body lists children under `## Section` headings, each entry as `- [[page-basename|Page Title]] — one-line summary`. The polish agent regenerates this list after every ingest; the agent's output is append-only (never deletes existing entries) so hand-curated groupings survive a regeneration pass.

**No `sources:` required.** Folder notes are navigation scaffolding, not knowledge claims. They are exempt from the `sources:` required check. They should not carry knowledge-content prose; that belongs on concept or entity pages.

## Examples

The file `wiki/agents/agents.md` is the folder note for the agents topic. Its frontmatter contains `type: index`, `aliases: ["agents", "Agents", "agent layer"]`, `children: ["[[orchestrator|Orchestrator]]", "[[ingest-agent|Ingest Agent]]"]`, and `child_indexes: []`. Its body has a `## Executor Agents` section listing each agent with a one-line summary.

Running `engine migrate --write` on a vault with `wiki/agents/_index.md` renames that file to `wiki/agents/agents.md` and rewrites any `[[_index]]` or `[[agents/_index]]` links to `[[agents|Agents]]`.

## Related Concepts

The folder note implements the per-folder MOC. It is the target of every non-root page's `parent:` field. The vault MOC at `wiki/index.md` is the root-level counterpart. Folder notes are created by the ingest agent when a new topic folder is opened and regenerated by the polish agent after each ingest. The `legacy-index-filename` lint check detects pre-v3 `_index.md` files.
